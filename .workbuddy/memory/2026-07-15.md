## 修复：ZS(1) 笔中枢框崩溃（ChartLevelLineStyle.forLevel 断言 level>=2）
- 现象：`flutter run -d windows` 启动即抛 `assertion 'level >= 2' is not true`，栈在 `_drawZsOnMainChart` 调 `ChartLevelLineStyle.forLevel(kn)`（kn=1=笔中枢=K0中枢，main.dart 默认勾选 `zs(1)`）。
- 根因：`forLevel` 原只服务连线/合并（内部 level≥2），ZS 指标合法值域含 level=1，越界。combine 路径对 kn==1/2 已特判不走 forLevel，故仅 ZS(1) 触发。
- 修复（`chart_level_line_style.dart`）：`assert(level >= 2)` → `assert(level >= 1)`；新增 `case/if level==1` 返回独立蓝靛色 `0xCC3B82F6`（不与 K1连线琥珀撞色），保持「ZS(n) 同层同色系」约定。
- 其余 forLevel 调用点（kline_chart.dart:1631 `_drawOneLevelLines` 有 `if(level<2) return` 守卫；:2225 峰值距路径 `labelKn==1` 提前 return）均不会以 level=1 进入，安全。
- 验证：`flutter analyze` 退出码 0（无新增错误，剩余为 test 文件 pre-existing lint）。用户需 R 热重启生效。
