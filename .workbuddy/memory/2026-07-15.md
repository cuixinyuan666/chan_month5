## CHAN_RUST 原生缠论中枢(ZS) 全层同构实现 + Flutter 消费（已落地、测试全绿）

### 背景与硬约束（来自用户）
- 实现完整原生中枢 + 保留 v1 跨段中枢并重命名(KuaDuan→KuaDuanV1) + 三类买卖点留到后续 pass。
- **代码全层同构**：每层用同一 `find_zs` 建立在 `LevelSegment` 上（K0=level1/K1=level2…）。
- **基于当前 Rust 版已实现的形态学元素**，禁止更改任何已实现元素逻辑（合并引擎/分型/段构造/层级递归/v1 算法）。
- 不引入 Python 式独立「笔(bi)」。

### 关键技术决策
- `find_zs` 只读冻结 `LevelSegment`（`self.segments` 排除 `active_unit`），天然无未来函数。
- 离开-返回延伸（区别于 v1 的 i=j 直接关闭）：离开段不重叠、返回段再重叠则延伸同一中枢，离开段被跳过（不计入 member_segs）。
- 九段重叠升级（member_segs>=9）、combine 合并（zs 模式比[ZG,ZD]/peak 比[DD,GG]）、one_bi_zs 单段中枢、normal/over_seg/auto 多算法。
- **Flutter 端 ZS 不做 Dart 本地重算**：pipeline `export()` 每步都基于冻结段重算 `zs_frames`（Rust 权威末态），Flutter 直接消费 `bundle.zsFrames`。这样避免 Dart/Rust 算法漂移，也贴合「基于 Rust 版形态学元素」约束。

### 背景与硬约束（沿用、未破）
- 复用前序 ZS 范式（同文件每次只改一处；Rust 只读冻结段 + 不碰已实元素逻辑；Flutter 直接消费 Rust 末态 `bsp_frames`，不本地重算）。
- **背驰策略（用户决策）：纯结构趋势末端，不做 MACD/力度背驰**——Rust 端无 MACD/metrics，避免引入指标依赖。

### 关键设计点
- 一类与二三类共用段结构：二三类均在一类端点之后的「离开段(p+1)+返回段(p+2)」上判定（二类看回踩深度、三类看 ZS 非回带），正确对齐原版缠论。
- 趋势分组：相邻、同向单调（下跌：后中枢 dd<前中枢 dd 且连接段 dir=-1；上涨镜像）的中枢串成 run；run 长度 ≥ min_zs_cnt 才判一类。
- ZS 带口径：zg=max(low)，zd=min(high)；三类买要求返回段 low>=zd，三类卖要求返回段 high<=zg。
- Flutter 渲染：买红/卖绿（涨红跌绿），一类圆/二类三角/三类菱形，旁注「买N/卖N + 价位」。
